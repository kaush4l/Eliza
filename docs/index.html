elizaOS Live — WASM edition

Debian-based elizaOS in your browser via container2wasm (Bochs x86_64). Default login: eliza / password: eliza (auto-typed on boot) — the greeter then offers Amnesia or Persistent storage.
First-time setup: connect an LLM (and enable Persistent mode)

The OS boots and logs in with no setup, but agent chat needs an OpenAI-compatible model server, and Persistent mode needs somewhere to store its sealed state. Both come from a small local bridge:

  1. Run a local OpenAI-compatible model server (LM Studio, llama.cpp server, vLLM, omlx…), e.g. gemma-4-12B-it-qat-mxfp8 at http://127.0.0.1:8873/v1.
  2. Download serve.py from the repo and run python3 serve.py 8901. Different model port: ELIZA_MODEL_UPSTREAM=http://127.0.0.1:PORT python3 serve.py 8901.
  3. Reload this page — it auto-detects the bridge on http://127.1:8901 and shows the backend in the status bar. (Or open http://127.0.0.1:8901/ directly to serve everything locally.)

Overrides: ?backend=<origin> any CORS-enabled bridge origin · ?model=<name> model id · ?autologin=0 type the login yourself.

In the greeter pick [1] Amnesia (nothing survives reload) or [2] Persistent (passphrase-sealed ~/.eliza, restored next boot). Chat commands: /save /quit /shell /log. First agent start under emulation takes several minutes.
