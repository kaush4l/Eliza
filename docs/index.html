Connecting your LLM

The OS boots and logs in on its own. Agent chat needs an OpenAI-compatible model server on this machine — LM Studio, Ollama, omlx, llama.cpp server…

  1. Start your model server with CORS allowed for this page. LM Studio: Developer tab → Enable CORS. Ollama: launch with OLLAMA_ORIGINS=*.
  2. Enter its base URL above (LM Studio http://127.0.0.1:1234/v1, Ollama http://127.0.0.1:11434/v1) and hit Test — it lists the models; pick one. If the browser asks to allow local network access, allow it.
  3. Done — the VM picks the URL up at boot. Changed it after the VM started? Reload the page.

This page is https, so the model server must be reachable via loopback on the machine running the browser (browsers exempt 127.0.0.1 from mixed-content rules; other hosts on your LAN are blocked).

State: boots default to Persistent mode — ~/.eliza is sealed (AES-256, auto-generated passphrase) and stored in this browser (Cache API), restored on the next boot. Nothing leaves your machine. Saving is not automatic: click Save state (or type /save) before closing/reloading, or recent chat is lost. Chat commands: /save /quit /shell /log.

URL overrides: ?backend=<url> · ?model=<name> · ?autologin=0 type the login yourself · ?storage=1 amnesia boot · ?storage=ask pick storage in the terminal.
